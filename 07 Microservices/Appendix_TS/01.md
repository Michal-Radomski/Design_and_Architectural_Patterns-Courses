Parameter properties in TypeScript are a shorthand syntax that allows defining and initializing class properties directly in
the constructor parameters by prefixing them with access modifiers like `public`, `private`, `protected`, or `readonly`. This
eliminates the need for separately declaring the property in the class body and then assigning it in the constructor, making
the code more concise.

For example, instead of writing:

```typescript
class Vehicle {
  public color: string;
  constructor(color: string) {
    this.color = color;
  }
}
```

You can write:

```typescript
class Vehicle {
  constructor(public color: string) {}
}
```

Here, `public color: string` in the constructor automatically declares and initializes the `color` property.

Key points:

- Parameter properties combine declaration and initialization into one step.
- They take effect by adding visibility (or readonly) modifiers to constructor parameters.
- It simplifies class code and reduces boilerplate.

This feature is especially useful for compact and clean class constructors.[1][2][3][4][5][6]
